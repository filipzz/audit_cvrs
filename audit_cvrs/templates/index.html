Audit CVRs Data Site

This site is powered by the audit_cvrs application, which is associated with the ElectionAudits software, which provides software and documentation in support of the Principles and Best Practices for Post-Election Audits and the other work of ElectionAudits.org, the nation's clearinghouse for election audit information.

The software is open source, still in very active "alpha" development

Audit steps

  1. Run the election, save Cast Vote Records (CVRs), and create a manifest of how many ballots are in each batch
  2. Publish the CVRs (or a commitment to them) for the general public.
  3. Set the audit up by using Tools for Comparison Risk-Limiting Election Audits and follow the directions to:
    1. Create a manifest showing which ballots are in which batches
    2. Have various stakeholders roll dice to generate a verifiably random seed to determine which ballots are audited
    3. Enter the contest parameters
    4. Draw a random sample of ballots to be audited
    5. Look up ballots in the manifest to create the "Sorted lookup table"
  4. Copy and paste the "Sorted lookup table" it provides into a file, and read it in to this tool using the "./manage.py parse file" command from the command line.
  5. Visit the list of selected CVRs via this link (in a separate browser tab if you like): CVR. You will need to log in the admin account you set up via manage.py. It will show the selected CVRs for the audit.
  6. Run through the given CVRs. Click each one:
    1. Assign it each for auditing, by setting the status to "assigned".
    2. Retrieve the proper ballot, write down the manual interpretation of the ballot, and compare the manual interpretation with the machine interpretation.
    3. Set the Discrepancy field to "Interpretations match" or to the proper discrepancy value, and set the status to "Completed".
